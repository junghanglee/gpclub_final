## Codebase Patterns

- Router pending UI is centralized in `src/router.tsx`.
- Static route framing should render immediately without waiting for remote data.
- Remote record/list/detail sections should use shared skeleton components while loading.
- Dynamic loading must not use fallback product, catalog, event, company, contact, FAQ, or product-image data to look populated.
- Empty states should appear only after the relevant loading flag is false.
- `LoadingModal` is a fullscreen overlay and should be reserved for true blocking flows.
